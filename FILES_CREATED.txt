===============================================================================
                            DATABASE SCHEMA
===============================================================================

New Tables:

1. pinned_messages
   - id (UUID) - Primary key
   - message_id (UUID) - References messages
   - room_id (TEXT)
   - pinned_by (TEXT) - User ID who pinned
   - pinned_at (TIMESTAMP)
   - UNIQUE(message_id, room_id)

2. user_blocks
   - id (UUID) - Primary key
   - blocker_id (TEXT) - User doing blocking
   - blocked_user_id (TEXT) - User being blocked
   - reason (TEXT) - Optional reason
   - created_at (TIMESTAMP)
   - UNIQUE(blocker_id, blocked_user_id)

3. notification_settings
   - id (UUID) - Primary key
   - user_id (TEXT) - UNIQUE
   - sound_enabled (BOOLEAN)
   - desktop_enabled (BOOLEAN)
   - email_enabled (BOOLEAN)
   - mute_until (TIMESTAMP)
   - muted_rooms (TEXT[]) - Array of room IDs
   - created_at, updated_at (TIMESTAMP)

4. notifications
   - id (UUID) - Primary key
   - user_id (TEXT) - Recipient
   - type (TEXT) - Notification type
   - actor_id (TEXT) - Who triggered it
   - actor_name (TEXT)
   - room_id (TEXT)
   - message_id (UUID)
   - title (TEXT)
   - content (TEXT)
   - read_at (TIMESTAMP)
   - created_at (TIMESTAMP)

5. analytics_snapshots
   - id (UUID) - Primary key
   - metric_type (TEXT)
   - metric_name (TEXT)
   - value (BIGINT)
   - metadata (JSONB)
   - created_at (TIMESTAMP)

Indexes Created:
  - idx_pinned_messages_room_id
  - idx_pinned_messages_message_id
  - idx_user_blocks_blocker_id
  - idx_user_blocks_blocked_user_id
  - idx_notifications_user_id
  - idx_notifications_created_at
  - idx_analytics_snapshots_metric_type
  - idx_analytics_snapshots_created_at

RLS Policies:
  - Read/write policies for all new tables
  - User data isolation
  - Admin-only analytics writes
